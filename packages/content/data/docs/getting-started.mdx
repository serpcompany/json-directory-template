# Getting Started

<p className="mt-2 text-xl text-muted-foreground">
  Get the starter running locally, point it at a checked-in site config, and
  ship a static build artifact.
</p>

## Before you start

- Use Node `24.13.1` from the repo `.nvmrc`
- Use `pnpm` `10`
- Decide which checked-in site you are building under `sites/<id>/`

## Install dependencies

```bash
nvm use
pnpm install
```

## Run the app locally

For normal UI work, use the web app only:

```bash
pnpm dev
```

That starts the active site wrapper on `http://localhost:3005`.

Use these variants when you need them:

- `pnpm dev:starter` runs the default starter wrapper directly
- `pnpm dev:all` runs the older full Turbo workspace loop
- `pnpm dev:site -- --site <id>` prepares the selected site's listing data and starts dev with the correct site env
- `pnpm --filter starter dev:inspect` starts the starter wrapper with the Node inspector attached

## Understand the checked-in site inputs

The active starter keeps site identity and shell behavior in checked-in config files:

- `sites/site-config.default.ts` defines the starter defaults
- `sites/<id>/site-config.ts` holds sparse per-site overrides
- `sites/<id>/site-content.ts` holds optional site-owned modules such as `externalResources` and `listingCliInstall`

The current listing source of truth is usually `data/listings.json`, though a site-specific build step can transform another JSON file into that shape before export.

## Build a static site artifact

Build one site at a time:

```bash
pnpm build:site -- --site your-site-id
```

The output lands under `dist/sites/<site-id>` based on the checked-in build config.

## Useful checks

Validate listing data before building:

```bash
pnpm validate:listings data/listings.json
```

If your site uses a source adapter like `sites/<id>/products.json`, regenerate the normalized dataset first:

```bash
pnpm prepare:site -- --site your-site-id
```

Run the main test suite:

```bash
pnpm test
```

Refresh the search index explicitly when you need to inspect the generated artifact:

```bash
pnpm generate-search-index
```
