# Common Commands

<p className="mt-2 text-xl text-muted-foreground">
  These are the commands you will use most while developing, validating, and
  exporting a site from the starter.
</p>

## Local development

Run just the web app:

```bash
pnpm dev
```

Run the older multi-workspace loop:

```bash
pnpm dev:all
```

Run one checked-in site with its data prep and env wired for you:

```bash
pnpm dev:site -- --site serpdownloaders.com
```

Run the web app with the Node inspector attached:

```bash
pnpm --filter web dev:inspect
```

## Validation and tests

Validate the active listing JSON shape:

```bash
pnpm validate:listings data/listings.json
```

Run the full repo tests:

```bash
pnpm test
```

Run focused web tests:

```bash
pnpm --filter web exec jest --runInBand --runTestsByPath app/docs/__tests__/page.test.tsx
```

## Build output

Build the workspace:

```bash
pnpm build
```

Build one static site artifact:

```bash
pnpm build:site -- --site your-site-id
```

Generate the search index directly:

```bash
pnpm generate-search-index
```

## Data generation helpers

Prepare the active normalized listing dataset for one site:

```bash
pnpm prepare:site -- --site serpdownloaders.com
```

Transform a site-specific JSON source into `data/listings.json`:

```bash
pnpm exec tsx scripts/trial-build.ts sites/your-site/products.json data/listings.json
```

Generate the search artifact after data changes:

```bash
pnpm exec tsx scripts/search-index-generator.ts
```

## Notes

- `pnpm dev` is the stable local default. The repo keeps `pnpm dev:all` for the heavier multi-workspace loop.
- `pnpm dev:site -- --site <id>` is the safest path when the site does not use the default checked-in config.
- The starter expects Node `24.x`.
- When you are working on one site, prefer `build:site` over the broader root `build`.
