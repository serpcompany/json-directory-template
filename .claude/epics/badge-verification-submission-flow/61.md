# Task: VerifyPage UI — badge preview, embed code, copy button, verify trigger

## Context

The `/submit/verify` page is where submitters complete verification. It mirrors the code.market "Verify & Publish" UX: numbered steps, badge embed code, light/dark toggle, and a prominent "Verify Now" button. Depends on the badge SVGs (59) being available and the submit API (60) so we can look up the submission from its token.

## Implementation

**Create `apps/web/app/submit/verify/page.tsx`:**

Page reads `?token=` from the URL, fetches submission details from `/api/submission?token=<token>`, and renders the verify flow.

**Layout (mirrors code.market UX):**

```
┌─────────────────────────────────────────┐
│  ← Back    Verify & Publish  ✓           │
├─────────────────────────────────────────┤
│  ① Add the badge to your site           │
│     [Light] [Dark]  ← toggle            │
│                                          │
│  ┌──────────────────────────────────┐   │
│  │ [Featured on {site}] ← preview   │   │
│  └──────────────────────────────────┘   │
│                                          │
│  <div data-token="abc123">...</div>      │
│  [Copy snippet]                          │
│                                          │
│  ② Click verify when badge is live      │
│  ┌──────────────────────────────────┐   │
│  │      ✓  Verify & Publish         │   │
│  └──────────────────────────────────┘   │
│                                          │
│  Troubleshooting...                      │
└─────────────────────────────────────────┘
```

**Embed snippet format:**

```html
<a href="https://{siteUrl}" title="Featured on {siteName}">
  <img 
    src="https://{siteUrl}/badge/featured-on-{siteId}-light.svg"
    alt="Featured on {siteName}"
    data-verify-token="{token}"
    width="153" height="44"
  />
</a>
```

The `data-verify-token` attribute is what the crawl endpoint scans for.

**Create `apps/web/app/api/submission/route.ts`** (`GET ?token=`) — returns submission name + website for the page to display. Does not expose full submission data.

**Components to create:**

- `BadgePreview` — renders `<img>` pointing to the static SVG URL, switches src on light/dark toggle.
- `CopySnippet` — textarea + copy button (uses `navigator.clipboard.writeText`).
- `VerifyButton` — calls `POST /api/verify-badge`, shows loading state, then success/error message.

**States:**

- `idle` → shows "Verify Now" button.
- `loading` → button shows spinner, disabled.
- `success` → green checkmark, "Your submission is verified! We'll review it soon."
- `error` → red message with reason + troubleshooting link.

## Acceptance Criteria

- [ ] `/submit/verify?token=abc123` loads and shows the submitter's name and website.
- [ ] Badge preview renders the correct SVG (light by default, dark on toggle).
- [ ] Embed snippet includes `data-verify-token="{token}"`.
- [ ] Copy button copies the full snippet to clipboard.
- [ ] "Verify Now" button calls `/api/verify-badge` and shows loading state.
- [ ] Success state shows a clear confirmation message.
- [ ] Error state shows the reason + troubleshooting steps.
- [ ] Invalid/missing token shows a 404-style message ("Submission not found").
