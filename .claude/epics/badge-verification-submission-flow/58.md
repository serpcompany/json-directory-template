# Task: Define submission JSON schemas and Zod types

## Context

The badge verification flow needs two new data files: `data/submissions-pending.json` and `data/submissions-verified.json`. These are the foundation for all other tasks — the submit API writes to pending, the verify API promotes to verified, and the admin CLI reads both.

## Implementation

**Create `sites/submission-schema.ts`:**

```ts
import { z } from 'zod'

export const PendingSubmissionSchema = z.object({
  token: z.string(),           // short hash: url-slug + timestamp
  name: z.string(),
  website: z.string().url(),
  category: z.string(),
  description: z.string(),
  submittedAt: z.string(),     // ISO 8601
  verifyAttempts: z.number().default(0),
})

export const VerifiedSubmissionSchema = PendingSubmissionSchema.extend({
  verifiedAt: z.string(),      // ISO 8601
  publishedAt: z.string().optional(),
})

export type PendingSubmission = z.infer<typeof PendingSubmissionSchema>
export type VerifiedSubmission = z.infer<typeof VerifiedSubmissionSchema>
```

**Create `data/submissions-pending.json`** (empty array `[]`) and **`data/submissions-verified.json`** (empty array `[]`).

**Add to `.gitignore`** (or document that these are committed per the static-first pattern):

- For MVP: commit them as empty arrays so the files exist; real data can be on a private branch.

**Token generation utility** in `apps/web/lib/submission-token.ts`:
```ts
import { createHash } from 'crypto'

export function generateToken(website: string): string {
  const slug = new URL(website).hostname.replace(/\./g, '-')
  const ts = Date.now().toString(36)
  const hash = createHash('sha256').update(website + ts).digest('hex').slice(0, 8)
  return `${slug}-${hash}`
}
```
