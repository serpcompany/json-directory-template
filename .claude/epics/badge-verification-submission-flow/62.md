# Task: POST /api/verify-badge — crawl submitter URL, check token, promote to verified

## Context

The core verification step. When a submitter clicks "Verify Now", this endpoint fetches their website's HTML server-side and scans for the presence of `data-verify-token="{token}"`. If found, the submission is promoted from pending to verified. If not found, returns a clear error with actionable troubleshooting.

## Implementation

**Create `apps/web/app/api/verify-badge/route.ts`:**

```ts
export async function POST(req: NextRequest) {
  const { token } = await req.json()
  if (!token) return NextResponse.json({ verified: false, message: 'Missing token' }, { status: 400 })

  // Load pending submissions
  const pending = await readSubmissions('pending')
  const submission = pending.find(s => s.token === token)
  if (!submission) return NextResponse.json({ verified: false, message: 'Submission not found' }, { status: 404 })

  // Rate limit check
  if (submission.verifyAttempts >= 5) {
    return NextResponse.json({ verified: false, message: 'Too many attempts. Contact support.' }, { status: 429 })
  }

  // Increment attempt count
  await writeSubmissions('pending', pending.map(s =>
    s.token === token ? { ...s, verifyAttempts: s.verifyAttempts + 1 } : s
  ))

  // Fetch submitter's page
  let html: string
  try {
    const res = await fetch(submission.website, {
      signal: AbortSignal.timeout(8000),
      headers: { 'User-Agent': 'DirectoryVerifier/1.0' },
    })
    html = await res.text()
  } catch (e) {
    return NextResponse.json({
      verified: false,
      message: "Couldn't reach your site. Make sure it's publicly accessible and try again.",
    })
  }

  // Check for token
  const tokenPresent = html.includes(`data-verify-token="${token}"`)
  if (!tokenPresent) {
    return NextResponse.json({
      verified: false,
      message: 'Badge not found on your site. Make sure the snippet is placed on a publicly visible page.',
    })
  }

  // Promote to verified
  const verified = await readSubmissions('verified')
  await writeSubmissions('verified', [...verified, { ...submission, verifiedAt: new Date().toISOString() }])
  await writeSubmissions('pending', pending.filter(s => s.token !== token))

  return NextResponse.json({ verified: true, message: 'Verified! Your submission is now in review.' })
}
```

**Edge cases to handle:**
- Redirect chains (follow up to 3 redirects via `fetch` default behavior).
- Large pages: read only first 500KB to avoid memory issues (`res.body` stream with size limit).
- HTTPS-only: if `http://` is submitted, try `https://` first.
- Token in an iframe or JS-rendered content: document in troubleshooting that the badge must be in the initial HTML (not rendered client-side only).

## Acceptance Criteria

- [ ] Returns `{ verified: true }` when `data-verify-token="{token}"` is present in fetched HTML.
- [ ] Returns `{ verified: false, message }` with meaningful error when not found.
- [ ] Returns `{ verified: false, message }` with timeout message if site is unreachable within 8s.
- [ ] Rate-limits to 5 attempts per token; returns 429 on excess.
- [ ] On success: submission removed from `submissions-pending.json` and added to `submissions-verified.json` with `verifiedAt`.
- [ ] Does not expose internal file paths or stack traces in error responses.
