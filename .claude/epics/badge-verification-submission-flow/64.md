# Task: Wire + integrate end-to-end — form → verify flow, smoke test

## Context

Final integration task. Confirms all pieces connect: form submits → token returned → verify page loads → badge snippet shown → verify button hits API → crawl succeeds → verified JSON updated → CLI publish works. Also ensures nothing regressed in the existing form or listing display.

## Implementation

**Integration checklist:**

1. **Form → API**: Confirm `SubmitForm` POSTs to `/api/submit` and navigates to `/submit/verify?token=…`.
2. **Verify page load**: Confirm page reads token from URL, fetches submission details, renders badge preview.
3. **Badge SVGs**: Confirm light and dark SVG assets exist at expected public URLs (`/badge/featured-on-{siteId}-light.svg`).
4. **Embed snippet**: Confirm snippet contains correct token, badge src, and site link.
5. **Verify API**: Manually test with a page that contains the token in its HTML (can use a local `<html>` fixture).
6. **CLI publish**: Confirm `pnpm submissions publish <token>` appends a valid listing.
7. **Regression**: Existing `/submit` form still works; existing listings still render.

**Write a smoke test script** `scripts/test-submission-flow.ts`:

```ts
// 1. POST /api/submit with test data → assert token returned
// 2. Read submissions-pending.json → assert submission present
// 3. Simulate badge presence: inject token into a local HTML fixture
// 4. POST /api/verify-badge with token → assert { verified: true }
// 5. Read submissions-verified.json → assert submission promoted
// 6. Run `publish <token>` → assert listing appended to listings.json
// 7. Clean up test data from JSON files
```

Run with: `pnpm test:submission-flow`

- How submitters complete verification
