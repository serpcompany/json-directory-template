# Task: POST /api/submit — form ingest, token generation, pending write

## Context

Replace the current form's GitHub redirect with an API call that stores the submission and returns a token. The existing `github-issue-submit-form.tsx` form is modified to POST to `/api/submit` instead of redirecting. On success, it navigates to `/submit/verify?token=<token>`.

Can run in parallel with task 59 (badge generation).

## Implementation

**Create `apps/web/app/api/submit/route.ts`** (Next.js App Router route):

```ts
import { NextRequest, NextResponse } from 'next/server'
import { PendingSubmissionSchema } from '@/sites/submission-schema'
import { generateToken } from '@/lib/submission-token'
import { readSubmissions, writeSubmissions } from '@/lib/submissions-store'

export async function POST(req: NextRequest) {
  const body = await req.json()
  
  // Validate input
  const parsed = z.object({
    name: z.string().min(1),
    website: z.string().url(),
    category: z.string().min(1),
    description: z.string().min(10),
  }).safeParse(body)
  
  if (!parsed.success) return NextResponse.json({ error: parsed.error }, { status: 400 })
  
  const token = generateToken(parsed.data.website)
  const submission = {
    token,
    ...parsed.data,
    submittedAt: new Date().toISOString(),
    verifyAttempts: 0,
  }
  
  // Check for duplicate token
  const pending = await readSubmissions('pending')
  if (pending.find(s => s.token === token)) {
    return NextResponse.json({ token }) // idempotent — return existing token
  }
  
  await writeSubmissions('pending', [...pending, submission])
  return NextResponse.json({ token })
}
```
